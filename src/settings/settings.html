How It Works

Sonto uses Retrieval-Augmented Generation (RAG) to answer your questions from saved snippets.

1
Capture
You highlight text on any webpage and save it via keyboard shortcut (Alt+Shift+C) or right-click context menu. Browser history is also synced automatically every 30 minutes.
2
Embed
Each snippet is converted into a vector embedding (a list of numbers representing its meaning). Sonto uses your API key to call one of these models:
text-embedding-3-small
OpenAI · 1536 dimensions
Used when an OpenAI key is configured. Fast, cheap (~$0.02 per 1M tokens), high quality.
text-embedding-004
Google · 768 dimensions
Fallback when only a Gemini key is configured. Free tier available.
3
Store
Embeddings and snippet text are stored locally in your browser's IndexedDB. Nothing leaves your device at this point.
4
Search
When you ask a question, your query is embedded with the same model. Sonto finds the 5 most relevant snippets using cosine similarity on the stored vectors.
5
Answer
The relevant snippets are sent as context along with your question to the selected chat model (OpenAI or Gemini). The AI generates an answer grounded in your saved data.

Privacy

How Sonto handles your data.

Data Collection

Sonto collects no personal data. There are no accounts, no analytics, no telemetry, and no tracking of any kind. The extension is fully open source.

Local Storage

All saved snippets, their vector embeddings, and your settings are stored locally in your browser using IndexedDB and chrome.storage.local. No data is sent to any Sonto server because there is no Sonto server.

API Keys

Your API keys are stored locally on your device. They are only transmitted to the corresponding provider's API endpoint for authentication. Sonto never reads, logs, or forwards your keys anywhere else.

Embeddings

When you save a snippet or sync browser history, the text is sent to your configured embedding provider (OpenAI or Gemini) to generate a vector representation. This is a one-time operation per snippet. The text is not stored by the provider beyond what their API terms specify.

AI Chat

When you ask a question in the sidebar, the query and relevant snippet context (up to 5 matched snippets) are sent to your selected AI provider. The provider generates a response and returns it. Review their privacy policies:

  • OpenAI Privacy Policy
  • Google AI Terms of Service
Browser History

Sonto reads your browser history (last 30 days) to automatically save page titles as searchable snippets. This data never leaves your browser except for the embedding API call. You can delete any history snippet from the sidebar at any time.

Your Responsibility

You are responsible for the data you save and the API keys you provide. If you save sensitive content and then ask a question about it, that content will be sent to your chosen AI provider. Consider this before saving confidential information.
